# Runtime Artifact Taxonomy

This document outlines the complete categorized taxonomy of all supported artifact kinds within the agent kernel registry. This taxonomy bridges standard cloud infrastructure with advanced AI cognitive architectures and reliability frameworks.

## 1. Operational & Infrastructure Artifacts
These are the foundational resources, workloads, and configurations managed by the kernel environment:
- `service`: Long-running application services.
- `worker`: Background processing units.
- `job`: One-off execution tasks.
- `cronjob`: Scheduled recurring tasks.
- `queue_topic`: Messaging queues for asynchronous communication.
- `stream_consumer`: Event stream processing units.
- `database`: Relational or document data stores.
- `cache`: In-memory data stores.
- `vector_index`: Specialized storage for AI embeddings.
- `object_store`: Binary large object storage.
- `policy_bundle`: Authorization and access policies (e.g., OPA).
- `secret_binding`: Secure credential references.
- `config_bundle`: Environment and application configurations.
- `ingress_route`: Network routing and API gateways.
- `certificate`: SSL/TLS and identity certificates.
- `observability_pipe`: Telemetry, logging, and tracing pipelines.
- `slo_policy`: Service Level Objective definitions.
- `deployment_unit`: Bundled deployable assets.
- `adapter`: Integration layers for external systems.
- `protocol_contract`: Formal interface definitions between components.

## 2. Cognitive & Behavioral Theories (Agent Psychology)
These artifacts define the internal mental models, decision-making capabilities, and behavioral traits of the hosted agents:
- `psychology`: Base psychological profiling and behavioral baselines.
- `theory_of_mind`: Mechanisms for agents to attribute mental states to themselves and others.
- `decision_strategy`: Frameworks for evaluating options and selecting actions.
- `cognitive_profile`: Internal state (personality, goals, constraints) driving an agent's dynamic knowledge expansion.
- `active_probe`: Empirical tests or micro-actions an agent executes to gather real-time data and update its beliefs.
- `epistemic_trust`: A model that maintains a dynamic trust ledger for information sources and mathematically scales belief updates.

## 3. Strategic & Interaction Theories
These artifacts govern how agents strategically interact with their environment, users, and other agents:
- `game_theory`: Mathematical models of strategic interaction and conflict resolution.
- `business_strategy`: High-level strategic planning and value optimization models.

## 4. Advanced Reliability & Architecture Frameworks
These represent formal engineering and cognitive science frameworks utilized to ensure predictable, stable, and fault-tolerant agent execution:
- `framework`: General, extensible architectural templates.
- `control_theory`: Application of Model Predictive Control and iterative look-ahead simulations.
- `information_foraging_theory`: Dynamic, evidence-based retrieval strategies (advanced RAG).
- `active_inference`: Implementations of the Free Energy Principle, maintaining belief states and minimizing surprise.
- `bdi_architecture`: Belief-Desire-Intention cognitive structures.
- `actor_critic`: Architectures utilizing centralized evaluation of decentralized actor actions.
- `dual_process_theory`: Implementation of explicit knowledge boundaries (Fast vs. Slow thinking).
- `organization_science`: Application of human organizational structures to multi-agent systems for coordination and failure prevention.
